458  // Ideally what we want here is a Dialog/Media Window. The problem is that these
459  // are two orthogonal discriminations of CGUIWindow and there wasn't a previous
460  // accounting for this possibility through the use of making CGUIWindow a
461  // virtual base class of the pertinent subclasses. So now we're left with
462  // no good solution.
463  //
464  // <strike>So here we're going to have the 'main' hierarchy (the one visible to SWIG)
465  // go the way intended - through WindowXML, but we're going to borrow dialog
466  // functionality from CGUIDialog by using it as a Mixin.</strike>
467  //
468  // jmarshall says that this class has no reason to inherit from CGUIMediaWindow.
469  // At some point this entire hierarchy needs to be reworked. The XML handling
470  // routines should be put in a mixin.
471  //
