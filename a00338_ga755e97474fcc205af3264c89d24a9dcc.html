bool KodiAPI::AddOn::General::GetSettingString ( const std::string &  settingName,
std::string &  settingValue,
bool  global = false 
)

Get a string settings value for this add-on.

Warning
For this function and others below is no access on Kodi about passwords, paths and network parts allowed! Text input elements allow a user to input text in various formats. The "label" attribute must contain an id from the language file that indicates which text to display for the input field. To get path from add-on can be from GetSettingString the settingValue __addonpath__ used.
Parameters
[in]settingNameThe name of the setting to get.
[in]settingValueThe value.
[in]global[opt] If set to true becomes Kodi itself asked about a setting (default is false)
Returns
true if successfull done

String type examples for add-on ./resources/settings.xml

  • text
    • Allow a user to enter one line of text.
      1 <setting id="addon.language" type="text" label="30523" default="en-US"/>
      • id="string" (required) - the name of the setting.
      • label="id" (required) - an id from the language file that indicates which text to display.
      • option="hidden"|"urlencoded" (optional)
        • if set to "hidden", each characters entered by the user will be obfuscated with an asterisks ("*"), as is usual for entering passwords.
        • if set to "urlencoded", each non-alphanumeric characters entered by the user will be "escaped" using XX encoding.
      • default="value" (optional) - the default value.
  • folder
    • Allow the user to browse for and select a folder.
      1 <setting id="iconpath" type="folder" source="files" label="30048" default="" />
      2 <setting id="download-folder" type="folder" label="32010"/>
      3 <setting id="image_path" type="folder" source="pictures" label="32012" visible="eq(-2,1)" default=""/>
      4 <setting label="32033" type="folder" id="folder" source="auto" option="writeable"/>
      • id="string" (required) - the name of the setting
      • label="id" (required) - an id from the language file that indicates which text to display.
      • source="auto"|"images"|... (optional, default="auto") - select a starting folder for the browse dialog.
        Source types Description
        music Include music related sources
        video Include video related sources
        pictures Include pictures related sources
        programs Include praograms related sources
        local Include local system related sources
      • value="value" (optional, default="") - the default value.
      • option="writeable" (optional, default="") - the user can be allowed to create and select new folders by setting this argument.
  • file
    • To select file by the user.
      1 <setting id="rtmp_binary" type="file" label="103" default="" />
  • ipaddress
    • To use setting for ip address with the related dialog.
      1 <setting id="hostip" enable="eq(-1,true)" type="ipaddress" label="32302" default="127.0.0.1" subsetting="true" />
      • id="string" (required) - the name of the setting.
      • label="id" (required) - an id from the language file that indicates which text to display.
      • default="value" (optional) - the default value.
  • fileenum
    • Display files in a folder as an enum selector.
      1 <-- Example: List sub-folders of the 'resources' folder for this add-on in the settings. --/>
      2 <setting id="myenum" type="fileenum" values="resources" mask="/"/>
      3 <setting id="trailer.ratingMax" label="32062" type="fileenum" values="$PROFILE/settings/ratings" default="MPAA" mask="/"/>
      • mask="value" - filter selectable files. Examples: "/" - display only folders. "*.txt" - display only .txt files.
      • option="hideext" - hide file extensions.
  • labelenum
    • Shows a spin control about given string values and use a string value from selection.
      1 <setting id="bitrate" type="labelenum" label="30003" default="Max" values="Min|Max|Select" />
      2 <setting id="trailer.quality" label="32067" type="labelenum" values="480p|720p|1080p" default="720p" />
      3 <setting id="scope" label="32092" type="labelenum" values="16:9|2.35:1|2.40:1" default="0" />
      4 <setting id="quality" type="labelenum" label="$LOCALIZE[622]" values="720p|480p" default="720p" />

Code example:

#include <kodi/api2/addon/General.hpp>
std::string bitrate = "Max";
if (!KodiAPI::AddOn::General::GetSettingString("bitrate", bitrate))
KodiAPI::AddOn::General::Log(LOG_ERROR, "Couldn't get 'bitrate' setting, falling back to default");
