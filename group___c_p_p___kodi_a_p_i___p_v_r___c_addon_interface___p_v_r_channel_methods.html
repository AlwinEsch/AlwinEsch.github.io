Detailed Description

Functions for Kodi to become available channels from PVR client

Remarks
Either bSupportsTV or bSupportsRadio is required to be set to true.
If a channel changes after the initial import, or if a new one was added, then the add-on should call KodiAPI::PVR::Trigger::ChannelUpdate()
Note
All functions defined here are Calls from Kodi itself to the addon from class KodiAPI::PVR::CAddonInterface

std::vector< PVR_CHANNEL > &  channels 
)
inlinevirtual

Request the list of all channels from the backend.

Parameters
[in]bRadioTrue to get the radio channels, false to get the TV channels.
[out]channelslist to add available channls defined with the PVR_CHANNEL structure
Returns
PVR_ERROR_NO_ERROR if the list has been fetched successfully. See PVR_ERROR for all possible values.
Remarks
If bSupportsTV is set to true, a valid result set needs to be provided for bRadio = false.
If bSupportsRadio is set to true, a valid result set needs to be provided for bRadio = true.
At least one of these two must provide a valid result set.
