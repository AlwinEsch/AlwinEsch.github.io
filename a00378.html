Control Rendering
C++ » 4. GUI

Detailed Description

Window control for rendering own parts

This rendering control is used when own parts are needed. You have the control over them to render direct OpenGL or DirectX content to the screen set by the size of them.

Alternative can be the virtual functions from t his been ignored if the callbacks are defined by the CControlRendering_SetIndependentCallbacks function and class is used as single and not as a parent class.

It has the header #include <kodi/api2/gui/ControlRendering.hpp> be included to enjoy it.

Here you find the needed skin part for a rendering control

Note
The call of the control is only possible from the corresponding window as its class and identification number is required.
