Detailed Description

Check mark control class (deprecated).

ControlCheckMark(x, y, width, height, label[, focusTexture, noFocusTexture, checkWidth, checkHeight, alignment, font, textColor, disabledColor])

Used on the XLink Kai Host dialog. Similar to Radio Button but does not have the button texture, and the check-mark is on the left.

Warning
Depreciated control!
These control are not recommended to be used inside skins and will be removed from the skinning engine at some point. Or may not work at all.
Note
This class include also all calls from Control
Parameters
[in]xinteger - x coordinate of control.
[in]yinteger - y coordinate of control.
[in]widthinteger - width of control.
[in]heightinteger - height of control.
[in]labelstring or unicode - text string.
[in]focusTexture[opt] string - filename for focus texture.
[in]noFocusTexture[opt] string - filename for no focus texture.
[in]checkWidth[opt] integer - width of checkmark.
[in]checkHeight[opt] integer - height of checkmark.
[in]alignment[opt] integer - alignment of label
  • Flags for alignment used as bits to have several together:
    Defination name Bitflag Description
    XBFONT_LEFT 0x00000000 Align X left
    XBFONT_RIGHT 0x00000001 Align X right
    XBFONT_CENTER_X 0x00000002 Align X center
    XBFONT_CENTER_Y 0x00000004 Align Y center
    XBFONT_TRUNCATED 0x00000008 Truncated text
    XBFONT_JUSTIFIED 0x00000010 Justify text
[in]font[opt] string - font used for label text. (e.g. 'font13')
[in]textColor[opt] hexstring - color of enabled checkmark's label. (e.g. '0xFFFFFFFF')
[in]disabledColor[opt] hexstring - color of disabled checkmark's label. (e.g. '0xFFFF3300')
Note
You can use the above as keywords for arguments and skip certain optional arguments.
Once you use a keyword, all following arguments require the keyword.
After you create the control, you need to add it to the window with addControl().

Example:

1 ...
2 self.checkmark = xbmcgui.ControlCheckMark(100, 250, 200, 50, 'Status', font='font14')
3 ...
